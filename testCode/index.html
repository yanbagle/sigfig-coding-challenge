Sigfig front end programming test

Coding front ends connected to REST apis is a huge component of being a front end developer at SigFig. This part of our interview process is meant to simulate part of an average day for our developers. Code this like you would at work, we want to see how you structure and organize your code to build a working application.

Expected functionality:

  1. Show a list of companies
  2. Show the details/full record of an existing company
  3. Create a new company
  4. Edit an existing company's record
  5. Show a list of people who work at a given company
  6. Show the details for a specific person
  7. Edit a person's record
  8. Delete a person record
  9. Create a new person, associating them to an existing company
  10. Bonus: make the site responsive
We have two utility methods to create some test data for you, accesible via HTTP GET: Create test companies | Import people for a given company
